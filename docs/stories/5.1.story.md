# Story 5.1: Pet Sitter Profile Creation & Management

**Epic:** 5 — Pet Sitter Profiles & Pay-per-Listing
**Status:** Draft

---

## Story

**As a** pet sitter,
**I want** to create a profile that showcases my services,
**so that** pet owners can find and contact me.

---

## Acceptance Criteria

1. Pet Sitter profile fields: full name, bio (max 500 chars), service area (city/region — up to 3 areas), services offered (multi-select: boarding, dog walking, day care, drop-in visits), rates (free text, max 200 chars, e.g., "€15/night boarding"), experience (years as integer), languages spoken (multi-select), care portfolio (photo upload: max 6 photos, S3 private bucket, CloudFront served).
2. Profile create/update endpoints: `POST /api/v1/sitter/profile` and `PUT /api/v1/sitter/profile`. Auth required, `pet_sitter` role only.
3. Public sitter profile: `GET /api/v1/sitters/{id}` — returns all profile fields above. Contact details (email, phone) are **never** returned. The CTA is a contact form (Story 5.1 AC4).
4. Contact form endpoint: `POST /api/v1/sitters/{id}/contact` — body: `{ name, email, message }`. Sends message to sitter via AWS SES (queued). No contact details in response. Rate-limited: 5 submissions per IP per hour.
5. SitterDashboard shell: profile editor tab + "Contact Submissions" tab listing received messages (name, date, message excerpt).
6. Profile completeness prompt on dashboard if required fields (name, bio, at least one service area, at least one service) are missing.

  ```php
  class SitterProfile extends Model
  {
      protected $fillable = [
          'user_id', 'full_name', 'bio', 'service_areas',
          'services', 'rates', 'experience_years', 'languages',
      ];

      protected $casts = [
          'service_areas'    => 'array',

          'languages'        => 'array',

      public function user(): BelongsTo
      {
          return $this->belongsTo(User::class);
      }

      public function portfolioPhotos(): HasMany
      {
          return $this->hasMany(SitterPortfolioPhoto::class)->orderBy('sort_order');
      }

      public function contactSubmissions(): HasMany
      {
          return $this->hasMany(SitterContactSubmission::class)->latest();
      }

      public function isProfileComplete(): bool
      {
          return !empty($this->full_name)
              && !empty($this->bio)
              && !empty($this->service_areas)
              && !empty($this->services);
      }
  }
  ```

  ```typescript
  export interface SitterProfile {
    id: number
    full_name: string
    bio: string | null
    service_areas: string[]
    services: ('boarding' | 'dog_walking' | 'day_care' | 'drop_in')[]
    rates: string | null
    experience_years: number | null
    languages: string[]
    portfolio_photos: { id: number; url: string }[]
    is_complete: boolean
  }

  export interface ContactSitterPayload {
    name: string
    email: string
    message: string
  }

  export const getSitterProfile = (id: number) => axiosClient.get<{ data: SitterProfile }>(`/sitters/${id}`)

  export const getMySitterProfile = () => axiosClient.get<{ data: SitterProfile }>('/sitter/profile')

  export const createSitterProfile = (payload: Partial<SitterProfile>) =>
    axiosClient.post<{ data: SitterProfile }>('/sitter/profile', payload)

  export const updateSitterProfile = (payload: Partial<SitterProfile>) =>
    axiosClient.put<{ data: SitterProfile }>('/sitter/profile', payload)

  export const uploadPortfolioPhoto = (file: File) => {
    const form = new FormData()
    form.append('photo', file)
    return axiosClient.post('/sitter/profile/photos', form, {
      headers: { 'Content-Type': 'multipart/form-data' },
    })
  }

  export const deletePortfolioPhoto = (photoId: number) => axiosClient.delete(`/sitter/profile/photos/${photoId}`)

  export const contactSitter = (sitterId: number, payload: ContactSitterPayload) =>
    axiosClient.post(`/sitters/${sitterId}/contact`, payload)

  export const getContactSubmissions = (page = 1) =>
    axiosClient.get('/sitter/contact-submissions', { params: { page } })
  ```

```php
use Tests\TestCase;

class SitterProfileTest extends TestCase

        parent::setUp();
        $this->sitter = User::factory()->create(['role' => 'pet_sitter']);
        $this->sitter->assignRole('pet_sitter');

    private function validProfilePayload(array $overrides = []): array
    {
        return array_merge([
            'full_name'     => 'Ana Constantin',
            'bio'           => 'I love pets and have 5 years of experience.',
            'service_areas' => ['Chișinău'],
            'services'      => ['boarding', 'dog_walking'],
        ], $overrides);

    public function test_sitter_can_create_profile(): void
    {
        $this->actingAs($this->sitter, 'sanctum')
            ->postJson('/api/v1/sitter/profile', $this->validProfilePayload())
            ->assertStatus(201)
            ->assertJsonStructure(['data' => ['id', 'full_name', 'services', 'is_complete']]);
    }

    public function test_sitter_can_update_profile(): void
    {
        SitterProfile::factory()->create(['user_id' => $this->sitter->id]);

        $this->actingAs($this->sitter, 'sanctum')
            ->putJson('/api/v1/sitter/profile', $this->validProfilePayload(['full_name' => 'Ana Updated']))
```

### §2 — Contact Form Security

The sitter contact form is the only public endpoint in the system that:

1. Accepts user-supplied `email` and sends it elsewhere (to the sitter)
2. Creates a DB record without authentication

Mitigations in place:

- Rate limiting (5/IP/hour) — blocks bulk abuse
- `email:rfc,dns` validation — rejects obviously fake domains
- Message min 10 chars — prevents trivial spam
- `sender_ip` stored for abuse reporting, NOT returned to sitter in API (internal audit use only)
- No sitter contact details in response — prevents email harvesting via the contact endpoint

The sitter's email is only used as the mail "To" address internally — it never appears in any API response.

### §3 — Portfolio Photo Storage Pattern

Portfolio photos follow the same S3 private bucket + CloudFront pattern as listing photos (Story 2.2):

- S3 key pattern: `sitters/{sitter_profile_id}/portfolio/{uuid}.{ext}`
- CloudFront URL derived from key: `config('cloudfront.url') . '/' . $s3_key`
- Magic bytes validation on upload
- `s3_key` never in any API response — only CloudFront URL returned

Max 6 photos enforced in `SitterPortfolioController::store()` before accepting the upload.
