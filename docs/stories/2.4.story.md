# Story 2.4: Browse & Search Listings

**Epic:** 2 — Breeder Profiles & Listings
**Status:** Approved

---

## Story

**As a** buyer,
**I want** to browse and filter pet listings,
**so that** I can find a pet that matches what I'm looking for without having to contact every breeder.

---

## Acceptance Criteria

1. Public endpoint `GET /api/v1/listings` returns paginated listings (20 per page): `{ id, title, breed, species, price, currency, location, main_photo_url, breeder_name, verified }`. No auth required. Contact details never included.
2. Filter parameters supported: `species`, `breed`, `location_city`, `location_region`, `price_min`, `price_max`, `verified_only` (boolean).
3. Default sort: featured listings first (`featured_until > NOW()`), then by `created_at` desc.
4. Keyword search: `?q=golden retriever` matches title, breed, description via MySQL FULLTEXT index.
5. Only listings with `status = 'active'` are returned.
6. Frontend renders a responsive grid; skeleton cards shown while fetching.
7. Active filters and `?q` persist in URL query params so filtered searches are shareable/bookmarkable.
8. Empty state displayed when no listing matches: "No listings match your search" with a "Clear filters" action.

- [ ] **T3** — Add `search` method to `ListingService`
  - File: `app/Services/ListingService.php` (update from Story 2.2)
  - Method: `search(array $filters): LengthAwarePaginator`
  - Query builder pattern:

    ```php
    public function search(array $filters): LengthAwarePaginator
    {
        $query = Listing::with(['breederProfile', 'photos' => fn($q) => $q->where('sort_order', 0)])
            ->where('status', 'active');

        // Keyword search via MySQL FULLTEXT
        if (!empty($filters['q'])) {
            $q = $filters['q'];
            $query->whereRaw('MATCH(title, description, breed) AGAINST(? IN BOOLEAN MODE)', [$q . '*']);
        }

        // Filters
        if (!empty($filters['species'])) {
            $query->where('species', $filters['species']);
        }
        if (!empty($filters['breed'])) {
            $query->where('breed', 'like', '%' . $filters['breed'] . '%');
        }
        if (!empty($filters['location_city'])) {
            $query->where('location', 'like', '%' . $filters['location_city'] . '%');
        }
        if (!empty($filters['location_region'])) {
            $query->where('location', 'like', '%' . $filters['location_region'] . '%');
        }
        if (!empty($filters['price_min'])) {
            $query->where('price', '>=', $filters['price_min']);
        }
        if (!empty($filters['price_max'])) {
            $query->where('price', '<=', $filters['price_max']);
        }
        if (!empty($filters['verified_only'])) {
            $query->whereHas('breederProfile', fn($q) => $q->whereNotNull('verified_at'));
        }

        // Sort: featured first, then newest
        $query->orderByRaw('CASE WHEN featured_until > NOW() THEN 0 ELSE 1 END')
              ->orderByDesc('created_at');

        $perPage = $filters['per_page'] ?? 20;
        return $query->paginate($perPage);
    }
    ```

  - NOTE: FULLTEXT requires MySQL 8 with full-text index on `listings(title, description, breed)` — created in Story 2.2 T15
  - LIKE on `breed` (in addition to FULLTEXT) for partial-match filtering without keyword context: when `q` is absent but `breed` filter is specified, FULLTEXT isn't used; LIKE provides straightforward substring match

## Dev Notes

### §1 — FULLTEXT vs LIKE: When Each is Used

| Scenario                  | Method                            | Why                           |
| ------------------------- | --------------------------------- | ----------------------------- |
| `?q=golden retriever`     | `MATCH...AGAINST` (FULLTEXT)      | Multi-word relevance matching |
| `?breed=retriever` (no q) | `WHERE breed LIKE '%retriever%'`  | Simple substring filter       |
| Both `q` and `breed`      | FULLTEXT on `q` + LIKE on `breed` | Combined precision            |

FULLTEXT `IN BOOLEAN MODE` with trailing `*` enables prefix matching (`golden*` matches "golden retriever"). This is sufficient for MVP. Elasticsearch or similar is not needed at this traffic level.

FULLTEXT index was added in Story 2.2 T15. If running tests on SQLite (in-memory), FULLTEXT is unsupported — use `config/database.php` to point tests to MySQL in `phpunit.xml`, or mock the FULLTEXT query in tests.

### §2 — Backend Feature Test Structure

```php
<?php

namespace Tests\Feature\Listing;

use Tests\TestCase;
use Illuminate\Foundation\Testing\RefreshDatabase;
use App\Models\User;
use App\Models\BreederProfile;
use App\Models\Listing;

class ListingSearchTest extends TestCase
{
    use RefreshDatabase;

    private BreederProfile $profile;

    protected function setUp(): void
    {
        parent::setUp();
        $breeder = User::factory()->create(['role' => 'breeder']);
        $this->profile = BreederProfile::factory()->create(['user_id' => $breeder->id]);
    }

    public function test_returns_only_active_listings(): void
    {
        Listing::factory()->create(['breeder_profile_id' => $this->profile->id, 'status' => 'active']);
        Listing::factory()->create(['breeder_profile_id' => $this->profile->id, 'status' => 'draft']);
        Listing::factory()->create(['breeder_profile_id' => $this->profile->id, 'status' => 'sold']);

        $this->getJson('/api/v1/listings')
            ->assertStatus(200)
            ->assertJsonCount(1, 'data');
    }

    public function test_filter_by_species(): void
    {
        Listing::factory()->create(['breeder_profile_id' => $this->profile->id, 'status' => 'active', 'species' => 'dog']);
        Listing::factory()->create(['breeder_profile_id' => $this->profile->id, 'status' => 'active', 'species' => 'cat']);

        $this->getJson('/api/v1/listings?species=dog')
            ->assertJsonCount(1, 'data')
            ->assertJsonPath('data.0.species', 'dog');
    }

    public function test_filter_verified_only(): void
    {
        $verified = BreederProfile::factory()->create([
            'user_id'     => User::factory()->create(['role' => 'breeder'])->id,
            'verified_at' => now(),
        ]);
        $unverified = $this->profile;  // no verified_at

        Listing::factory()->create(['breeder_profile_id' => $verified->id, 'status' => 'active']);
        Listing::factory()->create(['breeder_profile_id' => $unverified->id, 'status' => 'active']);

        $this->getJson('/api/v1/listings?verified_only=1')
            ->assertJsonCount(1, 'data')
            ->assertJsonPath('data.0.verified', true);
    }

    public function test_featured_listings_sorted_first(): void
    {
        Listing::factory()->create([
            'breeder_profile_id' => $this->profile->id,
            'status'             => 'active',
            'listing_type'       => 'standard',
            'featured_until'     => null,
            'title'              => 'Standard Listing',
        ]);
        Listing::factory()->create([
            'breeder_profile_id' => $this->profile->id,
            'status'             => 'active',
            'listing_type'       => 'featured',
            'featured_until'     => now()->addDays(7),
            'title'              => 'Featured Listing',
        ]);

        $this->getJson('/api/v1/listings')
            ->assertJsonPath('data.0.title', 'Featured Listing');
    }

    public function test_response_excludes_contact_details(): void
    {
        Listing::factory()->create(['breeder_profile_id' => $this->profile->id, 'status' => 'active']);

        $this->getJson('/api/v1/listings')
            ->assertJsonMissing(['phone'])
            ->assertJsonMissing(['email'])
            ->assertJsonMissing(['health_certificate_key'])
            ->assertJsonMissing(['s3_key']);
    }

    public function test_draft_listing_not_returned(): void
    {
        Listing::factory()->create(['breeder_profile_id' => $this->profile->id, 'status' => 'draft']);

        $this->getJson('/api/v1/listings')
            ->assertJsonCount(0, 'data');
    }

    public function test_pagination_returns_correct_page_size(): void
    {
        Listing::factory()->count(25)->create([
            'breeder_profile_id' => $this->profile->id,
            'status'             => 'active',

            ->assertJsonPath('meta.total', 25)
            ->assertJsonPath('meta.last_page', 2);
```

### §3 — URL State for Shareable Searches

Filters live in the URL (`/search?species=dog&price_max=500&verified_only=1`) so:

- Users can bookmark or share a filtered search
- Browser back button restores the previous filter state correctly
- No localStorage required for filter state

The React pattern: `useSearchParams()` reads the current URL params; all `onChange` callbacks call `setSearchParams(newParams)` which updates the URL without a full navigation. `useListings(params)` is called with the parsed param values, which are the React Query cache key — so changing a filter triggers a fresh fetch while hitting the cache if the same filter combo was used before.

---

### §4 — FULLTEXT Test Compatibility

MySQL FULLTEXT is not available in SQLite (which Laravel uses by default for testing). Two options:

1. **Preferred for MVP:** Configure `phpunit.xml` to use the MySQL test database:
   ```xml
   <env name="DB_CONNECTION" value="mysql"/>
   <env name="DB_DATABASE" value="pawtrust_test"/>
   ```
2. **Alternative:** Use `RefreshDatabase` with MySQL and a test database seeded before the test run.

The `test_keyword_search_matches_title()` and `test_keyword_search_matches_breed()` tests require MySQL. All other tests can run on SQLite. Mark the search tests with `@group mysql` to allow CI to run most tests without MySQL.
