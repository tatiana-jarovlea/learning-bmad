# Story 6.1: Full Romanian + Russian Internationalization

**Epic:** 6 — i18n, Polish & Launch Readiness
**Status:** Draft

---

## Story

**As a** user in Moldova or Romania,
**I want** the entire platform interface in my preferred language,
**so that** I can use PawTrust comfortably without encountering English text.

---

## Acceptance Criteria

1. All user-facing UI strings (labels, buttons, error messages, empty states, email subjects/bodies) available in both `ro` (Romanian) and `ru` (Russian). No English visible to end users in production.
2. Frontend: React i18next with locale files `public/locales/ro/translation.json` and `public/locales/ru/translation.json`. Language preference stored in `localStorage`. Default: browser locale detection — prefer `ro`, fallback to `ru`, then `ro` if undetected.
3. Backend: Laravel lang files `lang/ro/` and `lang/ru/` cover all API validation error messages and transactional email templates.
4. Language toggle (RO / RU) in the navigation bar — switches all UI text instantly without page reload.
5. All transactional emails sent in the user's preferred language (stored on the `users` table as `locale` — defaults to `ro`).
6. Date formats localized: `DD.MM.YYYY` for both locales (same in this region).
7. Currency displayed as EUR (`€3`, `€8`) throughout — no localization needed for currency format.
8. QA: full manual walkthrough of all 10 core user-facing screens in both languages documented in a QA checklist before launch sign-off.

  ```php
  class SetLocale
  {
      public function handle(Request $request, Closure $next): mixed
      {
          // Prefer authenticated user's locale, then Accept-Language header, then default
          if ($request->user()) {
              App::setLocale($request->user()->locale ?? config('app.locale'));
          } else {
              $locale = $request->header('Accept-Language');
              $supported = ['ro', 'ru'];
              $detected = collect($supported)->first(
                  fn($l) => str_contains($locale ?? '', $l),
                  config('app.locale')
              );
              App::setLocale($detected);
          }
          return $next($request);
      }
  }
  ```

  - Register in `app/Http/Kernel.php` (or `bootstrap/app.php` in Laravel 11) as a global API middleware

### §1a — Backend Feature Test Stubs (T5)

```php
<?php

namespace Tests\Feature\Locale;

use Tests\TestCase;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Illuminate\Support\Facades\Mail;
use App\Models\User;
use App\Models\BreederProfile;
use App\Models\Listing;
use App\Mail\BreederVerificationApprovedMail;

class LocaleResolutionTest extends TestCase
{
    use RefreshDatabase;

    public function test_authenticated_user_locale_used_for_api_responses(): void
    {
        $user = User::factory()->create(['locale' => 'ru']);
        $user->assignRole('buyer');

        // Submit an empty login to trigger a validation error in the user's locale
        $response = $this->actingAs($user, 'sanctum')
            ->postJson('/api/v1/listings', [])  // missing required fields
            ->assertStatus(422);

        // The validation error for a required field should use the 'ru' locale
        $errors = $response->json('errors');
        $this->assertNotEmpty($errors);
        // Spot-check: error text should be Russian, not English
        $firstError = collect($errors)->flatten()->first();
        $this->assertStringNotContainsStringIgnoringCase('required', $firstError,
            'Validation message should be in Russian, not English');
    }

    public function test_accept_language_header_used_for_unauthenticated_requests(): void
    {
        // Submit to a public endpoint with Accept-Language: ru
        $response = $this->withHeaders(['Accept-Language' => 'ru'])
            ->postJson('/api/v1/auth/login', [])  // no credentials — validation error
            ->assertStatus(422);

        $errors = $response->json('errors');
        $firstError = collect($errors)->flatten()->first();
        // Should be Russian, not English
        $this->assertNotNull($firstError);
        $this->assertStringNotContainsStringIgnoringCase('required', $firstError);
    }

    public function test_invalid_locale_falls_back_to_ro(): void
    {
        // Accept-Language with unsupported locale falls back to 'ro'
        $response = $this->withHeaders(['Accept-Language' => 'fr'])
            ->postJson('/api/v1/auth/login', [])
            ->assertStatus(422);

        $errors = $response->json('errors');
        $firstError = collect($errors)->flatten()->first();
        $this->assertNotNull($firstError);
        // RO validation message should not be English "required"
        $this->assertStringNotContainsStringIgnoringCase('required', $firstError);
    }

    public function test_validation_errors_returned_in_user_locale(): void
    {
        $roUser = User::factory()->create(['locale' => 'ro']);
        $ruUser = User::factory()->create(['locale' => 'ru']);
        $roUser->assignRole('buyer');
        $ruUser->assignRole('buyer');

        $roError = $this->actingAs($roUser, 'sanctum')
            ->postJson('/api/v1/listings', [])
            ->json('errors.title.0');

        $ruError = $this->actingAs($ruUser, 'sanctum')
            ->postJson('/api/v1/listings', [])
            ->json('errors.title.0');

        // Both should have errors, and they should differ in language
        $this->assertNotNull($roError);
        $this->assertNotNull($ruError);
        $this->assertNotEquals($roError, $ruError,
            'RO and RU validation messages should differ');
    }

    public function test_email_sent_in_user_locale_ro(): void
    {
        Mail::fake();

        $breeder = User::factory()->create(['role' => 'breeder', 'locale' => 'ro']);
        $admin   = User::factory()->create(['role' => 'admin']);
        $admin->assignRole('admin');
        $breeder->assignRole('breeder');
        $profile = \App\Models\BreederProfile::factory()->create(['user_id' => $breeder->id]);
        $vr = \App\Models\VerificationRequest::factory()->create([
            'breeder_profile_id' => $profile->id,
            'status'             => 'pending',
        ]);

        $this->actingAs($admin, 'sanctum')
            ->putJson("/api/v1/admin/verification-requests/{$vr->id}/review", [
                'status' => 'approved',
            ]);

        Mail::assertQueued(BreederVerificationApprovedMail::class,
            fn($mail) => $mail->locale === 'ro'
        );
    }
```

### §1 — Key Parity Enforcement

As the project grows, RO/RU key files can drift (a key added to RO but forgotten in RU). Add a test:

```php
// tests/Unit/LocaleKeyParityTest.php
public function test_ru_locale_has_all_ro_keys(): void
{
    $ro = Arr::dot(require lang_path('ro/mail.php'));
    $ru = Arr::dot(require lang_path('ru/mail.php'));
    $missing = array_diff(array_keys($ro), array_keys($ru));
    $this->assertEmpty($missing, 'Missing RU keys: ' . implode(', ', $missing));
}
```

Run this for each lang file. Failing this test before launch means a user would see a raw key string (e.g., `mail.verification.approved.subject`) instead of translated text.

### §2 — `locale` vs `language` Field

The `users` table uses `locale` (values: `'ro'`, `'ru'`) — established in Story 1.2. Laravel's `App::setLocale()` accepts these directly. Confirm the frontend passes `Accept-Language: ro` or `Accept-Language: ru` on API calls made for unauthenticated users. For authenticated users, the `SetLocale` middleware reads from `$user->locale`.
