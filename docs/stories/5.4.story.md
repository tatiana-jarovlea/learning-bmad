# Story 5.4: Featured Listing Upgrade

**Epic:** 5 ÔÇö Pet Sitter Profiles & Pay-per-Listing
**Status:** Draft

---

## Story

**As a** breeder with an active listing,
**I want** to upgrade my listing to featured,
**so that** it appears at the top of search results and attracts more buyers.

---

## Acceptance Criteria

1. Active listings (status = `active`, `is_featured = false`) can be upgraded via `POST /api/v1/listings/{id}/upgrade-to-featured`. Auth required, breeder role, breeder must own the listing.
2. If the listing was created as standard (paid ÔéČ3), the upgrade fee is ÔéČ5 (featured total ÔéČ8 minus ÔéČ3 already paid). If the listing was already created as featured (paid ÔéČ8 upfront), this endpoint returns 400 "Listing is already featured."
3. On payment success (Stripe webhook `checkout.session.completed` with `metadata.action = 'upgrade_to_featured'`): `listings.is_featured` set to `true`. Listing moves to top of default sort.
4. Featured listings display a "Featured" badge on listing cards in search results and on the listing detail page. Visually distinct from the Verified badge.
5. Featured status lasts for the remainder of the listing's active period ÔÇö no separate expiry beyond the listing's own lifecycle (30-day active window is post-MVP; at MVP listings don't expire, so featured is permanent unless the listing is deleted/deactivated).
6. BreederDashboard: active standard listings show "Upgrade to Featured (ÔéČ5)" CTA. Active featured listings show a "Featured" badge (no upgrade CTA).

### ┬ž1b ÔÇö Why `400` (Not `409`) for Already-Featured

AC2 specifies HTTP 400 for "Listing is already featured." This is intentional: the UI should never show the "Upgrade to Featured" CTA for a listing that is already featured ÔÇö the `is_featured` field on the listing response drives the UI state. A request to upgrade an already-featured listing is a **client programming error** (a bug in the frontend), not a state conflict a user might legitimately encounter. `400 Bad Request` signals this to developers.

By contrast, `409 Conflict` is used for state collisions that can occur due to race conditions or concurrent users (e.g., duplicate inquiries, duplicate reviews). The already-featured check does not fall into that category.

---

### ┬ž1 ÔÇö Upgrade Pricing Logic

The ÔéČ5 upgrade fee represents the differential between the standard listing fee (ÔéČ3) and the featured listing fee (ÔéČ8). This is enforced only in the Stripe Checkout Session price ÔÇö PawTrust does not store "how much was paid" on the listing record. The logic is:

- If `listing.listing_type === 'standard'` Ôćĺ eligible to upgrade Ôćĺ charge `config('stripe.prices.upgrade')` (ÔéČ5)
- If `listing.listing_type === 'featured'` Ôćĺ already paid ÔéČ8 Ôćĺ return 400

This is checked server-side in `ListingUpgradeController` before creating the Stripe session. There is no way for a breeder to manipulate the price from the frontend ÔÇö the backend determines the amount.

---

### ┬ž2 ÔÇö Webhook `action` Metadata Disambiguation

Both `activate_listing` and `upgrade_to_featured` events come through the same Stripe webhook endpoint. The `metadata.action` field on the Checkout Session distinguishes them. The webhook handler uses a `match` on `action` to dispatch to the correct `ListingPaymentService` method.

This is safer than trying to infer the action from `metadata.listing_type` alone ÔÇö if a new action is added in future (e.g., `renew_listing`), it is explicitly handled or falls through to the `default` no-op case.

---

### ┬ž3 ÔÇö Featured Sort Position

Featured listings sort before standard listings via `ORDER BY is_featured DESC, listed_at DESC`. This means:

- All `is_featured = 1` listings appear first, sorted by `listed_at` among themselves (most recently listed featured listings first)
- Then all `is_featured = 0` listings, also sorted by `listed_at`

This is a stable, deterministic sort ÔÇö no randomization or auction for position. If two breeders upgrade to featured, they both appear in the featured tier, sorted by when their listing was first posted.
