# Story 1.3: Login, JWT Auth & Logout

**Epic:** 1 — Foundation & Authentication
**Status:** Draft

---

## Story

**As a** registered user,
**I want** to log in with my email and password and log out to end my session,
**so that** I can securely access my account and protect it when I'm done.

---

## Acceptance Criteria

1. Login endpoint: `POST /api/v1/auth/login` — accepts `email` + `password`, returns Sanctum token + user object `{ id, name, email, role, locale }`.
2. Invalid credentials return 401 with a generic error message that does NOT specify whether email or password is wrong (prevents user enumeration).
3. Sanctum token expires after 24 hours (configured in Story 1.2 — verify still correct).
4. Logout endpoint: `POST /api/v1/auth/logout` (Bearer auth required) — deletes the current token from `personal_access_tokens`. Returns 200.
5. All protected API routes return 401 if no valid token is provided or if the token is expired/deleted.
6. Frontend stores JWT in Zustand auth store (persisted to `localStorage`). Axios interceptor attaches `Authorization: Bearer {token}` to all API requests.
7. Frontend clears token and redirects to `/login` on logout or on receiving a 401 response from any API call.
8. Rate limiting: max 10 login attempts per IP per minute. Returns 429 on breach with standard error envelope.
9. Can be verified locally: login → get token → call protected endpoint → logout → call same endpoint → 401.

- [ ] **T3** — Add `login` method to `AuthService`
  - File: `app/Services/AuthService.php`
  - Method: `login(array $credentials): array`
    ```php
    public function login(array $credentials): array
    {
        if (!Auth::attempt(['email' => $credentials['email'], 'password' => $credentials['password']])) {
            throw new AuthenticationException('Invalid credentials.');
        }
        $user = Auth::user();
        $user->tokens()->delete(); // revoke previous tokens on re-login
        $token = $user->createToken('auth_token')->plainTextToken;
        return ['token' => $token, 'user' => $user];
    }
    ```
  - `Auth::attempt()` returns false for wrong email OR wrong password — same codepath, no enumeration
  - Revoke previous tokens on re-login (prevents session accumulation)
  - **Note:** `$user->tokens()->delete()` revokes ALL tokens for the user, including tokens from other devices/sessions. This is acceptable for MVP (single-device assumption). Multi-device token management (revoke only the relevant token) is a post-MVP concern.

## Dev Notes

### §1 — Security: No Credential Enumeration

**Critical rule:** The login error response MUST be identical whether the email doesn't exist OR the password is wrong.

**DO:**

```json
{ "message": "Invalid credentials." }
```

**DO NOT:**

```json
{ "message": "No account found with this email." }
{ "message": "Password is incorrect." }
```

Both `Auth::attempt()` failure cases (wrong email, wrong password) hit the same `AuthenticationException` → same 401 response. [Source: architecture/security.md — Prevents user enumeration (OWASP A07)]

---

### §2 — Token Expiry Behaviour

Sanctum tokens are checked on every request. When a token expires (`expires_at < now()`), Laravel returns 401 automatically. The frontend 401 interceptor (T13) clears the store and redirects to `/login`.

Token expiry is set in `config/sanctum.php`:

```php
'expiration' => 60 * 24, // 24 hours in minutes
```

This was configured in Story 1.2 — verify it is set correctly before proceeding.

### §3 — Backend Feature Test Structure

```php
<?php

namespace Tests\Feature\Auth;

use Tests\TestCase;
use Illuminate\Foundation\Testing\RefreshDatabase;
use App\Models\User;

class LoginTest extends TestCase
{
    use RefreshDatabase;

    private User $user;

    protected function setUp(): void
    {
        parent::setUp();
        $this->user = User::factory()->create([
            'email'    => 'ion@example.com',
            'password' => bcrypt('password123'),
            'role'     => 'buyer',
        ]);
    }

    public function test_user_can_login_with_valid_credentials(): void
    {
        $response = $this->postJson('/api/v1/auth/login', [
            'email'    => 'ion@example.com',
            'password' => 'password123',
        ]);

        $response->assertStatus(200)
            ->assertJsonStructure(['data' => ['token', 'user' => ['id', 'name', 'email', 'role']]]);
    }

    public function test_login_fails_with_wrong_password(): void
    {
        $response = $this->postJson('/api/v1/auth/login', [
            'email'    => 'ion@example.com',
            'password' => 'wrongpassword',
        ]);

        $response->assertStatus(401)
            ->assertJson(['message' => 'Invalid credentials.']);
    }

    public function test_login_fails_with_nonexistent_email(): void
    {
        $response = $this->postJson('/api/v1/auth/login', [
            'email'    => 'nobody@example.com',
            'password' => 'password123',
        ]);

        // MUST return same message as wrong password — no enumeration
        $response->assertStatus(401)
            ->assertJson(['message' => 'Invalid credentials.']);
    }

    public function test_login_response_does_not_expose_password(): void
    {
        $response = $this->postJson('/api/v1/auth/login', [
            'email'    => 'ion@example.com',
            'password' => 'password123',
        ]);

        $response->assertJsonMissing(['password']);
    }

    public function test_user_can_logout(): void
    {
        $tokenStr = $this->user->createToken('test')->plainTextToken;

        $response = $this->withToken($tokenStr)
            ->postJson('/api/v1/auth/logout');

        $response->assertStatus(200);
        $this->assertDatabaseMissing('personal_access_tokens', [
            'tokenable_id' => $this->user->id,
        ]);
    }

    public function test_logout_requires_authentication(): void
    {
        $this->postJson('/api/v1/auth/logout')
            ->assertStatus(401);
    }

    public function test_me_endpoint_returns_authenticated_user(): void
    {
        $token = $this->user->createToken('test')->plainTextToken;

        $this->withToken($token)
            ->getJson('/api/v1/auth/me')
            ->assertStatus(200)
            ->assertJsonPath('data.email', 'ion@example.com');
    }

    public function test_protected_route_rejects_deleted_token(): void
    {
        $token = $this->user->createToken('test')->plainTextToken;
        $this->user->tokens()->delete();

        $this->withToken($token)
            ->getJson('/api/v1/auth/me')
            ->assertStatus(401);
    }

    /**
     * Requires CACHE_DRIVER=array in phpunit.xml or .env.testing.
     * The rate limiter uses the cache driver; 'array' ensures in-memory
     * accumulation across the 11 requests within a single test process.
     */
    public function test_login_rate_limiter_returns_429(): void
    {
        for ($i = 0; $i < 10; $i++) {
            $this->postJson('/api/v1/auth/login', [
                'email'    => 'ion@example.com',
                'password' => 'wrongpassword',
            ]);
        }

        $this->postJson('/api/v1/auth/login', [
            'email'    => 'ion@example.com',
            'password' => 'wrongpassword',
        ])->assertStatus(429);
    }
}
```

### §4 — Axios Interceptor: Avoid Infinite Redirect

The 401 interceptor must not redirect when the 401 comes FROM the login endpoint itself (that's an auth failure, not a session expiry):

```typescript
axiosClient.interceptors.response.use(
  (response) => response,
  (error) => {
    const isLoginEndpoint = error.config?.url?.includes('/auth/login')
    if (error.response?.status === 401 && !isLoginEndpoint) {
      useAuthStore.getState().clearAuth()
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)
```

### Manual Smoke Test

```bash
# 1. Register a buyer
curl -s -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Ion","email":"ion@test.com","password":"password123","password_confirmation":"password123","role":"buyer"}' \
  | jq '.data.token'

# 2. Login
TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"ion@test.com","password":"password123"}' \
  | jq -r '.data.token')
echo "Token: $TOKEN"

# 3. Call protected endpoint
curl -s http://localhost:8000/api/v1/auth/me \
  -H "Authorization: Bearer $TOKEN" | jq '.data.email'
# → "ion@test.com"

# 4. Logout
curl -s -X POST http://localhost:8000/api/v1/auth/logout \
  -H "Authorization: Bearer $TOKEN"
# → {"message":"Logged out successfully."}

# 5. Confirm token rejected
curl -s http://localhost:8000/api/v1/auth/me \
  -H "Authorization: Bearer $TOKEN"
# → 401
```
