# Story 4.4: Review Submission

**Epic:** 4 ŌĆö Inquiry System & Contact Reveal
**Status:** Approved

---

## Story

**As a** buyer,
**I want** to leave a review for a breeder after a transaction,
**so that** I can help other buyers make informed decisions about who to trust.

---

## Acceptance Criteria

1. Review submission endpoint: `POST /api/v1/breeders/{id}/reviews`. Auth required, buyer role only.
2. Request body: `{ rating: integer 1ŌĆō5, text: string (max 500 chars) }`.
3. One review per buyer per breeder enforced ŌĆö duplicate attempt returns 409 with `{ message: "You have already reviewed this breeder." }`.
4. Reviews are immediately visible on the breeder's public profile after submission ŌĆö no moderation queue at MVP. Admin deletion of reviews (e.g., for ToS violations) is a **post-MVP** capability and is **not delivered in this story** ŌĆö see Dev Notes ┬¦4.
5. Reviews displayed on the public breeder profile page: star rating (visual stars), review text, buyer first name + last initial (e.g., "Maria P."), date posted.
6. Breeder profile shows aggregate stats: average rating rounded to 1 decimal place, total review count.
7. BuyerDashboard: "Leave a Review" CTA appears on inquiry cards where the buyer has an inquiry with `contact_revealed` status. If a review has already been submitted for that breeder, the CTA changes to "Review Submitted" (non-clickable). Editing reviews is not supported at MVP.

    ```php
    public function submit(User $buyer, BreederProfile $profile, int $rating, string $text): Review
    {
        // 1. Buyer must have an inquiry with this breeder's listings (prevent spam reviews)
        $hasInquiry = Inquiry::where('buyer_id', $buyer->id)
            ->whereHas('listing', fn($q) => $q->where('breeder_profile_id', $profile->id))
            ->exists();

        if (!$hasInquiry) {
            abort(422, 'You must have an inquiry with this breeder before leaving a review.');
        }

        // 2. Check for duplicate
        $existing = Review::where('breeder_profile_id', $profile->id)
            ->where('buyer_id', $buyer->id)
            ->first();

        if ($existing) {
            throw new DuplicateReviewException('You have already reviewed this breeder.');
        }

        return Review::create([
            'breeder_profile_id' => $profile->id,
            'buyer_id'           => $buyer->id,
            'rating'             => $rating,
            'text'               => $text,
        ]);
    }
    ```

### ┬¦1 ŌĆö Inquiry Gate for Review Submission

A buyer must have submitted at least one inquiry for any of the breeder's listings before leaving a review. This prevents:

- Buyers who never contacted the breeder from leaving fake positive/negative reviews
- Drive-by review bombing

This is a soft gate (checked in the service, returns 422). It does NOT require the inquiry to be in a specific state ŌĆö `contact_revealed` (the only state at MVP) is sufficient. If a future "inquiry accepted/rejected" flow is added, the gate can be tightened to require an accepted inquiry.

---

### ┬¦2 ŌĆö Buyer Privacy in Reviews

On public endpoints, never expose:

- Full buyer name ŌĆö use `buyer_display_name` accessor (`"Maria P."`)
- Buyer ID ŌĆö prevents cross-referencing buyer identity across reviews
- Buyer email, phone ŌĆö obviously excluded

The `ReviewResource` enforces this. The `buyer_display_name` accessor on the `Review` model handles the name masking. There is no configuration to disable this ŌĆö it is hardcoded behavior for privacy compliance.

---

### ┬¦3 ŌĆö Aggregate Rating Calculation

For MVP, the average rating is computed in-memory from eager-loaded reviews (max 20 loaded in the profile response). This avoids a separate aggregate query while the review count is small.

When review counts grow large, this should be replaced with denormalized `reviews_avg_rating` and `reviews_count` columns on `breeder_profiles`, updated via a DB trigger or model observer. This optimization is explicitly deferred as a post-MVP performance story.

For the `review_summary` in `BreederProfileResource`:

- `average_rating`: `round(collect($reviews)->avg('rating'), 1)` ŌĆö `null` if no reviews
- `total_count`: `count($reviews)` ŌĆö `0` if no reviews

---

### ┬¦4 ŌĆö Admin Deletion (No Moderation Queue)

Reviews go live immediately. Admin can delete a review reported as inappropriate via an admin endpoint (not implemented in this story ŌĆö post-MVP admin story). The architecture supports this because reviews are not soft-deleted at MVP; a hard delete by admin is acceptable since there is no buyer expectation of review persistence after a ToS violation.
